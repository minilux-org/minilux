# Contributing to minilux

## Getting Started
- Fork the repository and create feature branches from the latest `main` branch.
- Run `cargo fmt` and `cargo clippy --all-targets --all-features` before opening a pull request.
- Add or update tests in `examples/` when behaviour changes.

## Pull Request Checklist
- Provide a concise summary of your change and any relevant issue numbers.
- Include reproduction or verification steps when fixing bugs.
- Ensure the CI pipeline passes before requesting review.

## Communication
- Draft pull requests are welcome for early feedback.
- Use clear commit messages so reviewers can follow your reasoning.
